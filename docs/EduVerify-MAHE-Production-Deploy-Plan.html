4. Phase A — Prepare production PostgreSQL

1Activate PIM Contributor on subscription MAHEBLR-Admission (Azure Portal → My roles → Activate).
2Open PostgreSQL server: Portal → search mahe-ci-doc-prd-psqlsvr01 → Overview. Confirm Status = Ready.
3Create application database: Left menu → Databases → + Add → Name: eduverify → Save.
Do not use the default postgres database for the application.
4Networking: PostgreSQL → Networking → Enable Allow public access from any Azure service within Azure to this server → Save.
(After go-live, infra may restrict to private endpoints per target architecture.)
5Build DATABASE_URL for production App Service. Password must be URL-encoded (@ → %40):
postgresql://psqladmin:YOUR_URL_ENCODED_PASSWORD@mahe-ci-doc-prd-psqlsvr01.postgres.database.azure.com:5432/eduverify?sslmode=require
Store this in Key Vault secret DATABASE-URL — not in email or git.

5. Phase B — Prepare production Blob Storage

1Portal → mahecidocprdstgacct01 → Containers.
2Create container eduverify-documents if missing. Public access level: Private.
3Store production storage connection string in Key Vault as AZURE-STORAGE-CONNECTION-STRING (or use Managed Identity + Blob Data Contributor role — preferred long-term).
Never use UAT storage (mahecidocuatstgacct01) on production App Service. UAT and Prod storage must remain separate.

Work with MAHE IT to add these secrets to MAHE-CI-DOC-PRD-KV01:

node -e "console.log(require('crypto').randomBytes(48).toString('hex'))"

8. Phase E — Deploy application code to Production

Option 1 — GitHub Actions (recommended)

1GitHub repo mahe-eduverify → Settings → Secrets → Actions.
2Add secret AZURE_WEBAPP_PUBLISH_PROFILE — download from MAHE-CI-DOC-PRD-APP01 → Overview → Download publish profile (enable SCM basic auth if blocked).
3Update .github/workflows/mahe-prod-deploy.yml → set AZURE_WEBAPP_HOST to Prod App Service Default domain from Overview.
4GitHub → Actions → Deploy MAHE Production → Run workflow → type deploy-prod to confirm.

Option 2 — Azure CLI from developer machine

az login
az account set --subscription "MAHEBLR-Admission"

cd eduverify-server
az webapp deploy \
  --resource-group MAHE-CI-DOC-PRD-RG01 \
  --name MAHE-CI-DOC-PRD-APP01 \
  --src-path . \
  --type zip

13. Rollback plan

ScenarioAction
Bad code deployGitHub → re-run previous successful workflow commit; or App Service → Deployment Center → redeploy previous package
Database issueRestore PostgreSQL from Azure backup (point-in-time); contact IT
Gateway/DNS issueRevert DNS to maintenance page; IT rolls back gateway config
Critical bug at go-liveDisable student access via gateway; keep admin access for fixes; communicate delay to admissions

14. Post go-live monitoring (first 72 hours)

  • Monitor /api/health every hour (automated or manual)
  • Application Insights → failures, response times, exceptions
  • App Service → CPU/memory; scale out if CPU > 70% sustained
  • PostgreSQL → connections, storage growth
  • Storage account → capacity (plan ~100 GB per 5k students)
  • Verification Cell hotline for login/upload issues
